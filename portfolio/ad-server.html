Ad Server React App

CSC 307: Intro to Software Engineering // Sept-Dec 2020

A caption for the above graphic.

The Project

I worked with a team of five Computer Science students to create a working app for iFixit's ad server. I was responsible for designing the low and mid fidelity prototypes, and implementing the "New Ad" component.

Our project went like this:

  • Learn about how ads and campaigns work.
  • Meet with iFixit technical director Miles as a client, to ask questions to refine the requirements for this app.
  • Create prototypes for the app.
  • Build the app using React, Node, and Firebase as the database.

We decided to use React for the front-end and Node for the back-end, using Firebase for the database. We split up into two teams of three to have a front-end and back-end team.

The Prototypes

I created rough sketches of the interface based on what we discussed as a team. I created two versions to bring to the team and discuss which options would be better. Once we agreed on an idea, I created a higher fidelity prototype on Figma.

Implementation

I was responsible of the "New Ad" modal, which also includes a preview of the Ad. I used the React Bootstrap Modal and Grids to create this component. It was quite exciting how I was able to make the preview pretty accurate to actual ads on the iFixit website.

Challenges

Some challenges I faced with this project:

  • My component initially made with CSS grid was not clean, so I redid it with React Bootstrap Grid and Modal components. The CSS grid version wasted one week of my work time.
  • Front-end team couldn't complete our code until the back-end finished the API.
  • The 5 week time constraint.

Looking Back

This was my first time building a functional app, so it was exciting to see my implementations working on a real app. I was also able to apply my prototyping skills to this team, and I think it saved us a lot of time by having a clear, unified vision before implementation.
