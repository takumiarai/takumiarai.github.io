The Project

I worked with a team of five Computer Science students to create a working app for iFixit's ad server. I was responsible for designing the low and mid fidelity prototypes, and implementing the "New Ad" component.

Our project roadmap:

  • Learn about how ads and campaigns work.
  • Meet with iFixit technical director Miles as a client, to ask questions to refine the requirements for this app.
  • Create prototypes.
  • Build an MVP of the app.

We decided to use React for the front-end and Node for the back-end, with a database on Firebase. We split up into a front-end and back-end team. I joined the front-end team to implement components and connect them to the backend.

Implementation

I was responsible for building the "New Ad" modal with a preview of the Ad using front-end data. I used the React library to create this component. The Ad preview turned out pretty good, with options to center or left align the contents. It closely reflects the Ads actually used on the iFixit website.

Challenges

Some challenges I faced with this project:

  • The limited five week development period.
  • I initially made the Add Modal component using CSS grid, which ended up being more code than I needed. I redid it with the Modal component and React Grids, which turned out much simpler and better looking.
  • Front-end developers couldn't connect to the backend until the back-end team finished the API.

Looking Back

This was my first time building a functional app, so it was exciting to see my implementations working on a real app. Although the skillsets of our team members varied, everyone was honest with their communication, so I had a smooth experience working with this team. I was also able to apply my prototyping skills to this team of Computer Science students, and I think that saved us a lot of time by solidifying a unified vision before implementing the app.
