iFixit Ad Server

Fall 2020 // CSC 307: Intro to Software Engineering

iFixit Ad Server React app prototype and implementation.

Skills: Color, Typography, Wireframing, Figma, GitHub, HTML, CSS, React.

Team: Myself + 5 Computer Science Students

Overview

We built an app for the iFixit marketing team to easily manage ad campaigns for their website. I was responsible for wireframing, Figma prototyping, and implementing the "New Ad" component in React.

Our overall project roadmap:

  • Learn about how ads and campaigns work.
  • Meet with the iFixit technical director as a client and conduct an interview to better understand the requirements for this app.
  • Create prototypes.
  • Build an MVP of the app.

For the web app we decided to use ReactJS for the frontend, NodeJS for the backend, and Firebase for the database. I was a part of our front-end team, responsible for implementing the components and connecting them to the backend using a REST API.

Challenges

Some challenges I faced with this project:

  • Initially using CSS grids to build my New Ad Modal was time-consuming. For cleaner code and better usability, I rebuilt it with React components and grids.
  • Not agreeing on the naming of pages and components made it difficult to communicate.
  • Front-end developers were not able to connect to the backend until the backend team finished the API.

Looking Back

This was my first time building a functional app, so it was exciting to see my implementations working on a real app. Although the skillsets of our team members varied, everyone was honest with their communication, so I had a smooth experience working with this team. I was also able to apply my prototyping skills to this team of Computer Science students, and I think that saved us a lot of time by solidifying a unified vision before implementing the app.
