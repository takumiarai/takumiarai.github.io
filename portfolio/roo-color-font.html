Color and Typography Design System

Roopairs Product Design // Sept-Dec 2020

Color palette and typeface of the redefined design system.

I built a Color and Typography Design System to resolve accessibility and usability issues. Through research, I learned about the technical requirements with colors and fonts, and how to create an effective design system using Figma.

The Problem

Our users were experiencing readability issues with the app. They struggled to see the interface under outdoor glare, and the smaller fonts were hard to read.

I researched online across many UX articles and discovered two things. The interface was not producing enough contrast, and our font Nunito did not perform well on screens.

The Solution

We set two specific goals to solve this problem:

  1. Generate a color system that will help fulfill color contrast standards.
  2. Generate a type system that improves brand identity and provides a better visual balance.

Requirements

To accomplish these goals, we had to intialize a design system. I researched online to learn about how to create an effective design system. Then I found Mailchimp and ServiceTitan's design system, where they provided a simple and purposeful documentation of their interface design. With this as my primary source of reference, I noted several things:

  • Colors should have meaning corresponding to color psychology.
  • Colors and types should include documentation of usages.
  • Colors should have a tint and shade.
  • The typography should include a documentation of its intended feel.
  • The design system should be created on Figma, for its useful cloud library feature.

Building a design system will make our product more consistent, accessible, and reliable. It will also greatly help the future team to have a more efficient and effective design process.

Color

Initally, I used Material design's color tool to generate an accessible set of colors. This color palette unfortunately did not get approved because they wanted a wider variety of colors including orange and yellow. Then we decided to use paletton.com to generate a color palette with a combination of a triad and tetrad color combo. For our gray colors, we used Material design's color set.

Typography

Rubik has always been the font of our logomark, but it was not used for our interface. I looked more into Rubik and found some advantages:

  • The monospace numerical characters give an analog aesthetic to the dollar values.
  • The variety of font weights allows for a better expression of hierarchy.
  • The higher x-height provides better legibility at smaller sizes.

Yay! We then prototyped several screens to check its readability. I used a Figma plugin to measure the color contrast as we developed the new screens.

The Result

Once the new color pallete and type system were approved, I created the design system on Figma. Now that we have a Figma team library with the new design system, the new color and type system are set in stone and the front-end team can start working updating the code base of styles.

Challenges

The biggest challenge with this project was in making design decisions with limited technical knowledge. Since I am not an experienced professional in color, I relied on online software to generate a color palette while making sure that it passes the contrast standards. Althought I gave my best effort in creating an effective color palette, I'm sure that some adjustments will have to be made before the app is officially deployed. However, I do believe this project was a confident starting point for our new design system.

Looking Forward

Starting here, we are continuing to work on the design system by setting up an atomic design system and creating components to be used on every new screen we design. I hope that my efforts made for this document will help the product grow in a good direction.
